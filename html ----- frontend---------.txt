Do Not Use Tables for Layouts

The Mistake: Using <table> tags to position your search inputs and buttons on the screen.
Why avoid it: Tables are strictly for displaying tabular data (like your final student results).
What to do instead: Use CSS Flexbox or Grid to design your form layout.

Avoid "Div Soup" (Non-Semantic Code)

The Mistake: Wrapping every single element in generic <div> tags (e.g., <div class="button">Search</div>).
Why avoid it: Screen readers for visually impaired users and search engine bots cannot understand what your page means if everything is a <div>.
What to do instead: Use semantic HTML tags. Use <form> for inputs, <button> for actions, <main> for the main content, and <th>/<td> inside your table.

Do Not Hardcode Data Rows

The Mistake: Manually typing out 5 or 10 dummy student rows (<tr><td>John Doe</td></tr>) directly into your index.html file.
Why avoid it: In a real job, data is dynamic. Hardcoding rows defeats the purpose of learning JavaScript and APIs.
What to do instead: Leave your <tbody> completely empty in HTML. Let your JavaScript file create and insert those rows dynamically when the data arrives.

Never Skip the alt Attribute on Images

The Mistake: Adding an image (like a school logo) using <img src="logo.png"> without an alt description.
Why avoid it: It breaks web accessibility (a major point employers check for). 
If the image fails to load, the user sees nothing.
What to do instead: Always include a descriptive alt tag: <img src="logo.png" alt="University Campus Logo">.

Avoid Inline Styles

The Mistake: Adding design rules directly inside your HTML tags, such as <table style="width: 100%; color: blue;">.
Why avoid it: It makes your HTML incredibly messy, hard to read, and difficult to debug or change later.
What to do instead: Keep your styles completely separate in your style.css file using class selectors.

Do Not Mix Up Button Types

The Mistake: Forgetting to add type="submit" to your form's search button, or using a generic <a> link styled as a button.
Why avoid it: Forms have native browser behaviors (like submitting when a user presses the "Enter" key). If you don't use the correct button type, these built-in accessibility features break.
What to do instead: Always use <button type="submit">Search</button> inside a <form>.
